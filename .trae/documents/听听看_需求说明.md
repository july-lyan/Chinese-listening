# 听听看 TīngTīngKàn - 中文听力训练小工具需求说明

## 1. 产品概述

**产品名称**：听听看 TīngTīngKàn

**产品定位**：面向学中文的外国人（HSK 1-4水平）的听力训练工具，通过播放中文音频描述，展示两张相似但有细节差异的图片，用户听完后选出正确的那张。

**核心价值**：
- 提供有趣、互动的听力练习方式
- 帮助用户提高中文听力理解能力
- 通过视觉辅助增强记忆效果
- 适合不同水平的学习者使用

**目标用户**：
- 正在学习中文的外国人
- HSK 1-4级水平的学习者
- 希望通过趣味方式提高听力的用户

## 2. 核心功能

### 2.1 难度选择
- **功能描述**：用户可以选择不同难度级别的听力练习
- **难度级别**：
  - 初级（绿色）：HSK 1-2，简单词汇和句型
  - 中级（橙色）：HSK 3，中等难度词汇和句型
  - 高级（红色）：HSK 4，较复杂词汇和句型
- **设计要求**：每个难度级别以卡片形式展示，包含标题、副标题和难度说明

### 2.2 答题系统
- **功能描述**：用户听取音频描述后，从两张图片中选择正确的答案
- **图片展示**：
  - 桌面端：左右并排展示
  - 移动端：上下排列展示
- **音频控制**：
  - 播放/暂停按钮
  - 支持重复播放
  - 显示播放状态

### 2.3 语音合成
- **功能描述**：将文本转换为语音进行播放
- **技术实现**：使用Web Speech API
- **语速设置**：
  - 初级：较慢语速
  - 中级：中等语速
  - 高级：正常语速

### 2.4 答题反馈
- **功能描述**：用户选择答案后，系统给出反馈
- **反馈内容**：
  - 正确/错误判断
  - 显示中文原文
  - 显示拼音
  - 显示英文翻译
  - 关键词高亮（错误时）
- **音效反馈**：
  - 正确：播放"叮咚"音效
  - 错误：播放"嗡嗡"音效

### 2.5 结算系统
- **功能描述**：完成所有题目后，显示结算页面
- **结算内容**：
  - 总分（正确题数/总题数）
  - 答题回顾（每道题的正确/错误状态）
  - 提供"再来一次"和"换个难度"按钮

### 2.6 数据管理
- **功能描述**：管理题库数据
- **数据结构**：
  - 题目ID
  - 中文文本
  - 拼音
  - 英文翻译
  - 图片A路径
  - 图片B路径
  - 正确答案（A或B）
  - 关键词（用于高亮）
  - 难度级别
- **题目数量**：每个难度级别3道题，共9道题

## 3. 用户流程

1. **进入首页**：用户打开应用，看到难度选择页面
2. **选择难度**：用户选择适合自己水平的难度级别
3. **开始答题**：系统加载对应难度的题目
4. **听取音频**：用户点击播放按钮听取中文描述
5. **选择答案**：用户从两张图片中选择正确的答案
6. **查看反馈**：系统显示答题结果和相关信息
7. **进入下一题**：用户点击"下一题"按钮进入下一道题
8. **完成练习**：完成所有题目后，系统显示结算页面
9. **重新开始**：用户可以选择"再来一次"或"换个难度"

## 4. 页面设计

### 4.1 整体风格
- **主色调**：暖黄色 (#FFF8E1)
- **辅助色**：绿色 (#4CAF50)、橙色 (#FF9800)、红色 (#F44336)
- **按钮风格**：圆角矩形，有轻微阴影
- **卡片风格**：白色背景，圆角，轻微阴影
- **字体**：无衬线字体，清晰易读

### 4.2 页面布局

#### 4.2.1 难度选择页面
- 页面标题："听听看 TīngTīngKàn"
- 副标题："选择难度级别"
- 三个难度卡片：
  - 初级（绿色）：包含"初级"标题、"HSK 1-2"副标题、"简单词汇和句型"描述
  - 中级（橙色）：包含"中级"标题、"HSK 3"副标题、"中等难度词汇和句型"描述
  - 高级（红色）：包含"高级"标题、"HSK 4"副标题、"较复杂词汇和句型"描述
- 卡片排列：桌面端横向排列，移动端纵向排列

#### 4.2.2 答题页面
- 顶部：进度条（显示当前题目/总题数）
- 中部：
  - 播放按钮（中央位置）
  - 图片A和图片B（左右或上下排列）
- 底部：无（或简单导航）

#### 4.2.3 反馈页面
- 顶部：正确/错误提示（绿色对勾或红色叉号）
- 中部：
  - 中文原文（带拼音）
  - 英文翻译
  - 关键词高亮（红色）
- 底部："下一题"按钮

#### 4.2.4 结算页面
- 顶部："练习完成！"标题
- 中部：
  - 总分显示（如"8/9 正确"）
  - 答题回顾卡片（显示每道题的正确/错误状态）
- 底部："再来一次"和"换个难度"按钮

### 4.3 响应式设计
- **桌面端**：宽屏布局，图片左右并排
- **平板端**：适配中等屏幕，保持良好布局
- **移动端**：窄屏布局，图片上下排列，确保触摸友好

## 5. 技术要求

### 5.1 前端技术
- **HTML5**：页面结构
- **CSS3**：样式和响应式设计
- **JavaScript**：交互逻辑
- **Web Speech API**：语音合成
- **Web Audio API**：音效生成

### 5.2 数据存储
- **本地存储**：使用JavaScript对象存储题库数据
- **数据文件**：单独的data.js文件存储所有题目数据

### 5.3 性能要求
- **加载速度**：页面加载时间<3秒
- **响应速度**：用户操作响应时间<1秒
- **音频播放**：无明显延迟

### 5.4 兼容性
- **主流浏览器**：支持Chrome、Firefox、Safari、Edge
- **移动端**：支持iOS和Android设备
- **降级方案**：在不支持Web Speech API的浏览器中提供友好提示
